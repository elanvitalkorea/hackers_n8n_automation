* **개발의 정석:** 기획부터 배포까지 서비스가 흘러가는 큰 그림을 학습합니다.

* **확장 가능한 자동화:** n8n과 MCP(Model Context Protocol)를 활용해 단순 자동화를 넘어 '지능형 에이전트 시스템'을 구축합니다.

## 1-2. 바이브 코딩(Vibe Coding)과 Agentic AI (20분)

### 1. Product Engineer의 시대

AI 시대에는 기획자, 디자이너, 마케터, 개발자의 경계가 허물어지고 있습니다. 한 사람이 AI 도구를 활용해 여러 역할을 동시에 수행하는 **Product Engineer**가 인정받는 시대입니다.

* **변화:** "코딩을 평생 업으로 할 사람"이 아니더라도, 기본적인 개발 개념과 자동화 역량은 필수 소양이 되었습니다.
* **목표:** 전업 개발자가 되는 것이 아니라, 각자의 위치에서 AI와 코드를 도구처럼 자유자재로 활용하는 능력을 기릅니다.

### 2. 바이브 코딩(Vibe Coding)이란?

> "내가 만들고 싶은 서비스의 느낌(Vibe), 흐름, 규칙을 자연어로 설명하면, AI가 코드를 작성하는 방식"

* **전통적 코딩 vs 바이브 코딩:** 문법 하나하나를 타이핑하는 방식에서, 기획과 판단에 집중하고 구현은 AI에게 위임하는 방식으로 진화했습니다.
* **LLM이 코딩에 강한 이유:**
1. **방대한 데이터:** 수많은 오픈소스 코드로 학습되어 베스트 프랙티스를 알고 있습니다.
2. **구조적 적합성:** 코드는 논리적이고 규칙적인 구조를 가지므로 수학/논리에 강한 LLM이 잘 수행합니다.
3. **즉각적 검증:** 실행 결과를 통해 "성공/실패" 피드백 루프를 확실하게 돌릴 수 있습니다.



### 3. Agentic AI의 4대 요소

바이브 코딩과 자동화 시스템의 핵심에는 **Agentic AI** 구조가 있습니다. 우리가 사용할 Cursor나 n8n 워크플로우도 이 구조 위에서 동작합니다.

* **LLM (Brain):** 기본 두뇌 역할을 하는 거대 언어 모델입니다. 상황을 이해하고 판단합니다.
* **Autonomy (자율성):** 스스로 반복 루프를 돌며 "다음에 무엇을 할지" 결정합니다.
* **Memory (기억):** 과거 대화나 외부 정보를 저장해 두었다가 판단에 활용합니다.
* **Tools (도구):** 검색, DB 조회, 이메일 전송, 코드 실행 등 실제 작업을 수행하는 손과 발입니다.

### 4. 바이브 코딩 3단계 방법론

AI에게 일을 시킬 때도 체계적인 접근이 필요합니다.

1. **아이디어 정리:** "누구를 위해, 어떤 문제를 해결하며, 어떤 결과가 나와야 하는가?"를 **3문장**으로 정의합니다.
2. **스펙 작성 (PRD):** 필요한 화면, API, 예외 처리 등을 구체적인 글로 작성합니다.
3. **구현:** 정리된 내용을 바탕으로 Cursor(AI)에게 코딩을 지시합니다.

## 1-3. 자동화 도구 및 업무 선정 기준 (10분)

### 1. 자동화 도구 생태계

* **No-Code / Low-Code:**
* **n8n:** **(본 강의의 핵심)** 오픈소스 기반으로 커스터마이징이 강력하며, 복잡한 로직 제어가 가능합니다.
* **Zapier:** 가장 대중적이며 연동 앱이 많지만 유료 과금이 부담될 수 있습니다.
* **Make:** 시각적인 드래그 앤 드롭이 강점입니다.


* **Framework (개발자용):** LangChain, LangGraph, CrewAI 등 에이전트 개발을 위한 다양한 프레임워크가 존재합니다.

### 2. 어떤 업무를 자동화(Agent Job)해야 할까?

모든 일을 자동화할 수는 없습니다. HubSpot의 결정 트리에 따르면 다음 조건을 만족할 때 **Agent Job**으로 적합합니다.

* [x] **반복적이고 시간이 많이 드는가?**
* [x] **구조화된 데이터와 명확한 절차가 있는가?**
* [x] **정답률이 100%가 아니어도 허용되는가?** (치명적인 의사결정 제외)

## 1-4. [실전 팁] 문제 해결 검색 방법론 (15분)

개발 중 마주치는 에러를 해결하고 정보를 수집하는 효율적인 루틴입니다. 이 습관을 들이는 것이 바이브 코딩의 첫걸음입니다.

### 1. 화면/텍스트 수집 (OCR 활용)

에러 메시지나 화면의 텍스트를 AI에게 전달할 때는 **이미지가 아닌 텍스트**로 변환하는 것이 가장 정확합니다.

* **Mac:** Raycast (Screen OCR 확장) 권장
* **Windows:** PowerToys (`Win + Shift + T`로 텍스트 추출) 권장

### 2. 인프라 및 환경 문제 해결

설치나 환경 설정 문제는 일반 LLM보다 특화된 접근이 필요합니다.

* **Claude Code:** Docker, 서버 세팅 등 인프라 가이드를 받을 때 가장 성능이 좋습니다. "현재 OS + 에러 로그 + 최종 목표"를 명확히 제시하세요.
* **운영체제:** 가능하면 Mac 환경을 추천하며, Windows 사용자는 **WSL(리눅스 서브시스템)** 사용을 권장합니다.

### 3. 정보 정리 (Markdown & Obsidian)

* **Markdown:** LLM이 가장 잘 이해하는 포맷입니다. 질문과 답변을 Markdown 형식으로 정리하는 습관을 들이세요.
	* ChatGPT/ Gemini 결과 복사 버튼 활용 
	![[Pasted image 20251207102325.png]]
* **Obsidian:** 
	* 프로젝트별 폴더와 태그를 활용해 지식 베이스(Second Brain)를 구축합니다.
	* Web Scraper 웹 브라우저 확장으로 웹 문서도 함께 저장

### 4. 컨텍스트 확장 도구 

프로젝트의 전체 구조나 긴 문서를 AI에게 이해시킬 때 사용하는 도구들입니다.

* **Repomix:** 프로젝트 전체 코드를 하나의 텍스트 파일로 덤프하여 LLM에게 "이 프로젝트 구조를 파악해 줘"라고 할 때 사용합니다.
* **Sitefetch:** 온라인 문서를 통째로 긁어와 학습시킬 때 사용합니다.

```bash
# Sitefetch 사용 예시 (터미널)
npx sitefetch https://docs.n8n.io/ -o n8n-docs.md

```

- **Cursor 문서 추가 기능**
    - 프로젝트 관련 문서를 Cursor 셋팅으로 추가하여 referenc로 쓰기
        
- **Gemini Gem / ChatGPT Projects**
    - 프로젝트별로 지침과 문서를 지정하여 특정 TASK 수행
    - 특히 필요 지식이 있는 경우 매번 첨부하지 않아도 되어 편리함

### 5. 프롬프팅 3원칙

AI에게 질문할 때는 다음 3가지를 지켜야 원하는 답을 한 번에 얻을 수 있습니다.

1. **배경 설명:** "지금 어떤 프로젝트를 하고 있고, 무엇을 하려다 막혔는지" 구체적으로 서술.
2. **구분자 사용:** 코드나 로그는 백틱 3개로 감싸고, 문맥이 바뀔 때는 `>>>` 등으로 구분.
3. **단계별 요청:** 상황 설명 → 원인 추정 → 해결책 제안 → 코드 수정 순으로 단계를 나누어 질문.


### 6. 외부 검색 & 커뮤니티 활용
